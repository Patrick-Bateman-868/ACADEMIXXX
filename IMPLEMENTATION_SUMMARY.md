# Реализация по плану — что сделано

## Шаг 1. Аккаунты и вход
- **Регистрация** (`/signup`): email, имя, пароль (≥6 символов). После регистрации — автоматический вход и переход в профиль.
- **Вход** (`/login`) и **выход** (`/logout`). Сессии через Flask-Login.
- **Профиль** (`/profile`): только для залогиненных; редактируются имя, университет, страна, навыки, цели, интересы, ссылки. Email не редактируется.
- В шапке: для гостей — «Войти» и «Присоединиться», для пользователей — «Мой профиль» и «Выйти».
- В модели `User`: `password_hash`, `role` (user/admin). Первый зарегистрированный пользователь получает роль `admin`.

**Проверка:** зайти на `/signup`, зарегистрироваться, убедиться, что в шапке «Мой профиль» и «Выйти». Открыть `/profile` — форма с вашими данными.

---

## Шаг 2. Публичный профиль
- **Каталог** (`/profiles`): карточки студентов, поиск и фильтры (без изменений).
- **Просмотр профиля** (`/profiles/<id>`): страница студента в режиме только чтения. Для себя — кнопка «Редактировать профиль».
- В каталоге кнопка «Профиль →» ведёт на `/profiles/<id>`.

**Проверка:** открыть `/profiles`, перейти в профиль любого пользователя. Под своим профилем должна быть кнопка «Редактировать профиль».

---

## Шаг 3. Возможности (расширение)
- В модели **Opportunity** добавлены поля: `requirements`, `source` (источник/партнёр).
- В форме «Добавить возможность» — поля «Требования» и «Источник / партнёр».
- На странице возможности отображаются требования и источник; при залогиненном пользователе email в форме заявки подставляется автоматически.
- Категории: Грант, Стажировка, Проект/Работа, Ментор, Жильё (как в задании).

**Проверка:** добавить возможность с требованиями и партнёром, открыть карточку — всё видно. Войти и открыть форму заявки — email уже указан.

---

## Шаг 4. Поиск и фильтры
- **Поиск** по возможностям: по названию, описанию, требованиям, источнику.
- **Фильтры:** категория, «Только верифицированные», «Только с дедлайном».
- **Сортировка:** «Сначала новые» и «По дедлайну».
- Параметры фильтров сохраняются в URL и в пагинации.

**Проверка:** на `/opportunities` задать поиск, категорию, «С дедлайном», сортировку по дедлайну — список и пагинация работают с учётом фильтров.

---

## Шаг 5. Комьюнити — группы
- Модели **Group** и **GroupMember**. При первом запуске создаются 4 группы: AI & Data Science, Research & Academia, Entrepreneurship, Scholarships Abroad.
- **Сообщество** (`/community`): список групп. Отображается, в каких группах вы состоите.
- **Страница группы** (`/community/<id>`): описание, число участников, кнопки «Вступить» / «Выйти из группы» (для залогиненных).

**Проверка:** зайти в «Сообщество», открыть группу, нажать «Вступить» — появляется «Вы в группе» и кнопка «Выйти из группы».

---

## Шаг 6. Админ-панель
- **Роль admin:** первый зарегистрированный пользователь создаётся с ролью `admin`.
- **Админка** (`/admin`): доступна только при `current_user.role == 'admin'`. В шапке у админа отображается ссылка «Админ».
- На странице: таблицы «Пользователи» и «Возможности» с кнопками «Верифицировать» для пользователей (статус → verified) и возможностей (verified → True).

**Проверка:** зайти под первым пользователем (админом), в шапке есть «Админ». Открыть `/admin`, верифицировать пользователя и возможность — статусы меняются.

---

## Шаг 7. Полировка UI
- В навигации кнопка «Присоединиться» выделена золотым (primary).
- Бейдж «Verified» с золотым акцентом (цвет и рамка).
- Стили для админ-таблиц и адаптив (мелкий шрифт и отступы на узких экранах).

## Как запустить
```bash
pip install -r requirements.txt
python app.py
```
Откройте http://127.0.0.1:5000 . При первом запуске создаются таблицы, при необходимости — недостающие колонки и группы по умолчанию.

## Назначение админа вручную
Если нужно выдать роль admin другому пользователю (например, после того как первый уже создан), в SQLite:
```sql
UPDATE user SET role = 'admin' WHERE email = 'your@email.com';
```
